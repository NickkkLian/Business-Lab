# 选题与项目实验室 · Lab

一个纯前端的个人数据库：**商业化项目** + **自媒体方向**。
网页本身放在这个**公开仓库**（用 GitHub Pages 托管），你的数据存在另一个**私有仓库**里。
运行时网页用你的 GitHub Token 直接读写私有仓库——**Token 只保存在你自己浏览器的 localStorage，不会进入公开代码**。

```
┌─ 公开 repo（本仓库） ──────┐        ┌─ 私有 repo（你的数据） ─┐
│ index.html  ← GitHub Pages │  读写  │ business.json      │
│ README.md                  │ ─────► │ （提交历史 = 自动备份） │
└────────────────────────────┘  API   └─────────────────────────┘
```

## 一、部署网页（公开仓库）

1. 把 `index.html` 放进一个公开仓库。
2. 仓库 **Settings → Pages → Source 选 `main` 分支 / 根目录**，保存。
3. 等一两分钟，访问 `https://<你的用户名>.github.io/<仓库名>/`。

> 公开仓库里**只有界面代码**，没有任何数据和密钥，可以放心公开。

## 二、准备数据仓库（私有仓库）

1. 新建一个**私有**仓库（比如你现有的那个）。
2. 不需要手动建文件——首次保存时网页会自动创建 `business.json`。
   （也可以把本仓库的 `business.json` 拷过去当初始模板，路径随意，在设置里填一致即可。）

## 三、生成 Token（推荐 Fine-grained）

GitHub → **Settings → Developer settings → Fine-grained tokens → Generate new token**：

- **Repository access**：只选你的那个私有数据仓库
- **Permissions → Contents：Read and write**
- 过期时间按需设置

生成后复制 `github_pat_…`。

## 四、连接

打开网页 → 右上 **⚙ 设置 → 仓库连接**，填：

| 字段 | 示例 |
|---|---|
| 用户名/组织 | `your-name` |
| 私有仓库名 | `my-data` |
| 数据文件路径 | `business.json` |
| 分支 | `main` |
| Token | `github_pat_…` |

点 **连接并载入**。之后编辑内容 → 右上 **保存**（或 `Cmd/Ctrl+S`）即提交到私有仓库。

## 功能速览

- **四个大类**（平行）：商业化项目、自媒体方向、Freelance、备选职业；每类各有独立的评分维度
- **三种处置分类**：感觉好玩 / 有点说法 / 优先考虑，**拖拽即可改分类**
- **自定义评分维度 + 权重**，「越低越好」的成本类维度自动反向计入，综合分 0–100
- 每类都有专属长文区；自媒体三层分类：处置 › 视频/图文 › 具体类型（类型可增删）
- **研究资料库（独立 tab）**：把通用/相近的盈利模式、行业研究、博主拆解写成一份，四个大类都可**引用**它；每份研究有完整整页详情，并显示「被谁引用」
- 状态、标签、备注、搜索/排序/筛选
- **自动同步**：编辑即自动写回私有仓库，无需手动保存；顶栏状态徽标显示 已同步 / 同步中 / 待同步 / 失败，点它可立即同步或重试
- 同步稳健：自动处理 sha 冲突、失败自动退避重试、Token 失效会明确提示
- 项目 ↔ 自媒体方向交叉关联；概览页各类评分榜
- 导入/导出 JSON；GitHub 提交历史 = 免费的版本备份

## 隐私说明

- Token 仅存于浏览器 localStorage，不上传、不写入代码。
- 换设备需重新填 Token。
- 公共电脑用完请在设置里清空 Token。
